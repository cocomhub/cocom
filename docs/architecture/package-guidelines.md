# 包结构与依赖规范

## 目录分层
- pkg/: 面向外部的公共库代码。稳定、文档完备、可被其他模块依赖。
- internal/: 仅限本模块内部使用的实现细节。任何 external 路径均不得引用。
- cmd/: 可执行程序入口与装配层，组合 pkg 与 internal，避免业务逻辑下沉到 cmd。
- examples/: 示例与用法演示，演示对 pkg 的正确使用方式。

## 命名与导出规范
- 最小导出：默认不导出（小写标识符），仅当确有外部复用价值再导出。
- 文档注释：所有导出符号必须具备完整注释，首句简述功能并给出用例或链接。
- 弃用策略：标记 Deprecated: 注释并在变更日志说明去向；保留一个小版本过渡期。

## 依赖约束
- 减少跨包耦合：优先依赖更抽象/稳定的上层边界，避免横向依赖蔓延。
- 禁止循环依赖：包之间不得形成环；特别是 pkg 子包之间需保持有向无环。

## 公共 API 稳定性
- pkg/ 遵循 semver：破坏性变更仅在主版本（MAJOR）提升；新增特性在次版本（MINOR）。
