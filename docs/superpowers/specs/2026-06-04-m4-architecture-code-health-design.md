## 1. 统一错误响应格式

### 目标
消除代码中三种不同错误响应格式并存的问题，统一为 `{head, body}` 多层格式，新增标准错误码枚举。

### 规范

**标准错误码枚举**（新增 `pkg/httpwrap/errcode.go`）：

```go
package httpwrap

type ErrCode int

const (
    ErrCodeUnknown   ErrCode = -1   // 未知错误
    ErrCodeInvalid   ErrCode = -2   // 参数无效
    ErrCodeNotFound  ErrCode = -3   // 资源不存在
    ErrCodeForbidden ErrCode = -4   // 无权限
    ErrCodeInternal  ErrCode = -5   // 内部错误

    // 业务错误码保留原有值（1000, 1001, 2001 等不变）
)
```

**目标格式（已由 `httpwrap.GinRespondError` 实现）：**

```json
{
  "head": { "code": -2, "msg": "参数无效：缺少图片名称", "request_id": "xxx", "time": "..." },
  "body": null
}
```

### 改造点

| 层 | 当前 | 改为 |
|---|---|---|
| **View 层** `picture.go`, `gallery_detail.go` 等 | `c.AbortWithError(http.StatusBadRequest, err)` | `httpwrap.GinRespondError(c, 400, ErrCodeInvalid, "描述信息")` |
| **Server 层** `server.go` 部分直接响应 | `c.JSON(200, gin.H{"success": false})` | `httpwrap.GinRespondError(c, 500, ErrCodeInternal, "消息")` |
| **旧版 handler** `/api/*` | `httpwrap.ResponseFail(w, msg)` | 保持不动（已使用 httpwrap 格式） |
| **新版 handler** `/v2/api/*` | `httpwrap.GinRespondError(w, code, msg)` | 升级到使用 `ErrCode` 常量 |

### 不包含的范围
- 旧版 `/api/` 路由的响应格式不做改动（保持 `httpwrap.Response*`）
- 不引入 Swagger / OpenAPI 文档

## 3. 配置文档自动生成

### 核心机制

`//go:generate` → 扫描 `// config-doc:` 注释 → 对比 `viper.SetDefault` 和 `viper.Get*` → 输出 Markdown。

### 注释规范

在 `viper.SetDefault` 调用的上一行添加 `// config-doc:` 注释：

```go
// config-doc: cocom.storage.path | string | /data/cocom/data/gallery | 漫画图片存储目录
viper.SetDefault("cocom.storage.path", "/data/cocom/data/gallery")

// config-doc: log.enableFile | bool | false | 是否启用文件日志
viper.SetDefault("log.enableFile", false)

// config-doc: server.cors.enabled | bool | false | 是否启用 CORS
viper.SetDefault("server.cors.enabled", false)
```

格式：`// config-doc: <key> | <type> | <default> | <description>`

### 生成脚本

`tools/config-doc-gen/main.go`：

- 递归扫描 `cmd/`、`internal/`、`pkg/` 下的所有 `.go` 文件（排除 `*_test.go` 和 `tools/`）
- 匹配两种模式提取：
  - **已知键**：有前导 `config-doc` 注释的 `viper.SetDefault` → 输出文档表格
  - **未文档化键**：`viper.SetDefault("xxx")` 但无 `config-doc` 注释 → 在"未文档化"区域列出
  - **缺失默认值键**：`viper.Get*` / `viper.UnmarshalKey` 读取但无对应的 `viper.SetDefault` → 在"MISSING DEFAULT"区域列出
- 按配置段（log / mongo / server / comic 等）分组输出 Markdown 表格
- 文档末尾追加版本标记：`<!-- generated at {timestamp} | commit: {hash} -->`

### 触发方式

新建 `internal/config/gen.go`：

```go
package config

//go:generate go run ../../tools/config-doc-gen/main.go --output ../../docs/config.md
```

运行 `go generate ./internal/config/...` 即生成。

### Makefile 集成

```makefile
config-doc:
	@cp docs/config.md docs/config.md.bak 2>/dev/null || true
	go generate ./internal/config/...
	@echo "=== 配置变更对比 ==="
	@-diff docs/config.md.bak docs/config.md 2>/dev/null || true
	@rm -f docs/config.md.bak
	@echo "=== config-doc generated ==="
```

### 缺失默认值补充

初次生成时一并补齐以下键的 `viper.SetDefault` + `config-doc` 注释：

| 键 | 读取位置 | 建议默认值 | 类型 |
|---|---|---|---|
| `server.listen.http.addr` | `cmd/server/server.go:99` | `:35456` | string |
| `server.shutdown_timeout` | `cmd/server/server.go:160` | `5s` | duration |
| `admin.token` | `cmd/server/server.go:70` | `""` | string |
| `debug.allow_remote` | `pkg/middlewares/localguard.go:15` | `false` | bool |
| `admin.allow_remote` | `cmd/server/view/init.go:52` | `false` | bool |
| `http.enable_proxy` | `pkg/download/downloader.go:148` | `false` | bool |
| `http.proxy` | `pkg/download/downloader.go:149` | `""` | string |
| `client.server_addr` | `cmd/gallery.go:22` | `http://localhost:35456` | string |
| `archive.root_dir` | `internal/archivecli/commands.go:52` | `""` | string |

## 5. Makefile 统一

```makefile
test:
	go test -race ./...

	golangci-lint run || echo "golangci-lint not available, skipping"
```

```makefile
lint:
	golangci-lint run ./... || echo "golangci-lint not available, skipping"
```

- 不引入 tool installation 到各项目
- 不修改 `make build` / `make fmt` 等已有目标
